## 高频考点
- 为什么索引快：B+树时间复杂度O(logN)，减少磁盘I/O
- 索引分类：主键/唯一/普通/全文；聚簇/非聚簇；B+树/Hash
- 聚簇索引 vs 非聚簇索引：聚簇索引叶子节点存数据，非聚簇存主键（回表）
- 回表：非聚簇索引查到主键后，再用主键查聚簇索引获取完整数据
- 覆盖索引：查询列全在索引中，无需回表
- 最左前缀原则：联合索引(a,b,c)，查询必须包含a才能用到索引
- 索引失效场景：like '%xx'、函数、类型转换、or、!=、is null/not null
- 为什么用 B+ 树不用 B 树：B+树叶子节点有指针，范围查询快
- 为什么不用红黑树：红黑树高度太高，B+树矮胖减少磁盘I/O
- 为什么不用 Hash：Hash不支持范围查询和排序
