## 🎯 图片命名规范

为了保证可扩展性，采用了清晰的命名规范：

### 命名格式

```
assets/{笔记文件名}/{描述性名称}.{扩展名}
```

### 示例

- `assets/4-TCP协议详解/tcp-header.webp` - TCP 头部格式
- `assets/4-TCP协议详解/tcp-handshake.jpg` - TCP 三次握手
- `assets/3-HTTPS与加密/tls-handshake.png` - TLS 握手流程

### 优势

1. **目录对应**：每个笔记文件有自己的图片文件夹
2. **描述清晰**：文件名一看就知道是什么图片
3. **易于管理**：添加新图片时知道放在哪里
4. **避免冲突**：不同笔记的同名图片不会冲突

## 📝 如何添加新图片

### 步骤 1：确定图片所属笔记

例如：想给 `2-HTTP协议.md` 添加新图片

### 步骤 2：保存图片到对应文件夹

```bash
# 下载图片
cd 计算机网络/assets/2-HTTP协议/
curl -o http-cache.jpg "https://example.com/cache.jpg"
```

### 步骤 3：在笔记中引用

```markdown
![HTTP 缓存机制](assets/2-HTTP协议/http-cache.jpg)
```

### 步骤 4：如果是新建笔记

```bash
# 1. 创建图片文件夹
mkdir -p assets/6-新笔记名称

# 2. 下载图片
cd assets/6-新笔记名称
curl -o image.jpg "https://..."

# 3. 在笔记中引用
# ![描述](assets/6-新笔记名称/image.jpg)
```

## ⚙️ Obsidian 配置建议

### 设置附件文件夹（可选）

如果图片还是不显示，可以在 Obsidian 中配置：

1. 打开 **设置（Settings）**
2. 进入 **文件与链接（Files & Links）**
3. 设置 **附件默认存放路径（Default location for new attachments）**：
   - 选择 "In subfolder under current folder"
   - 子文件夹名称：`assets`

### 图片链接格式

Obsidian 支持两种格式：

**Markdown 标准格式**（当前使用）:
```markdown
![描述](assets/文件夹/图片.jpg)
```

**Obsidian Wiki 格式**（备选）:
```markdown
![[assets/文件夹/图片.jpg]]
```

如果标准格式不显示，可以尝试 Wiki 格式。

## 🔧 故障排除

### 问题 1：Obsidian 中图片不显示

**解决方法**：

1. 检查路径是否正确
2. 确认图片文件存在
3. 尝试重启 Obsidian
4. 检查 Obsidian 设置中的附件路径配置

### 问题 2：想修改图片存放位置

**方法**：

```bash
# 移动整个 assets 文件夹
mv assets attachments

# 批量替换笔记中的路径
sed -i '' 's|assets/|attachments/|g' *.md
```

### 问题 3：图片太大

**压缩方法**：

```bash
# 使用 ImageMagick 压缩（需先安装）
brew install imagemagick

# 压缩 JPG（80% 质量）
mogrify -quality 80 assets/**/*.jpg

# 压缩 PNG
mogrify -strip assets/**/*.png
```
