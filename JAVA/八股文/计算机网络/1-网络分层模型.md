## OSI 七层模型

### 🔥 一句话速答（⭐⭐⭐）
物理 → 数据链路 → 网络 → 传输 → 会话 → 表示 → 应用

### 📊 七层详解

| 层次 | 名称 | 作用 | 协议/设备 | 数据单位 |
|------|------|------|-----------|----------|
| **7** | **应用层** | 提供网络服务接口 | HTTP、FTP、SMTP、DNS | 报文 |
| **6** | **表示层** | 数据格式转换、加密压缩 | ASCII、JPEG、加密 | 报文 |
| **5** | **会话层** | 建立、管理、终止会话 | RPC、SQL | 报文 |
| **4** | **传输层** | 端到端可靠传输 | TCP、UDP | 段（Segment） |
| **3** | **网络层** | 路由选择、寻址 | IP、ICMP、路由器 | 包（Packet） |
| **2** | **数据链路层** | 帧同步、差错控制、MAC 地址 | 以太网、交换机、网桥 | 帧（Frame） |
| **1** | **物理层** | 比特流传输、物理介质 | 双绞线、光纤、集线器 | 比特（Bit） |

### 📝 各层详细说明

#### 7. 应用层（Application Layer）

**作用**: 最靠近用户，提供网络服务

**常见协议**:
- **HTTP/HTTPS**: 网页浏览
- **FTP**: 文件传输
- **SMTP**: 邮件发送
- **POP3/IMAP**: 邮件接收
- **DNS**: 域名解析
- **SSH**: 安全远程登录

**示例**: 浏览器、邮件客户端

#### 6. 表示层（Presentation Layer）

**作用**: 数据格式转换、加密、压缩

**功能**:
- 数据编码转换（ASCII ↔ EBCDIC）
- 数据压缩（GZIP）
- 数据加密（SSL/TLS）

**示例**: JPEG 图片编码、视频编码

#### 5. 会话层（Session Layer）

**作用**: 管理会话，控制对话

**功能**:
- 建立会话令牌
- 维护会话状态
- 终止会话

**示例**: RPC（远程过程调用）、SQL 会话

#### 4. 传输层（Transport Layer）

**作用**: 端到端通信，保证数据完整性

**协议**:
- **TCP**: 面向连接、可靠
- **UDP**: 无连接、不可靠、快速

**功能**:
- 分段与重组
- 流量控制
- 错误检测与纠正

#### 3. 网络层（Network Layer）

**作用**: 路由选择，跨网络传输

**协议**:
- **IP**: 寻址、路由
- **ICMP**: 控制消息（ping）
- **ARP**: IP 地址 → MAC 地址
- **RARP**: MAC 地址 → IP 地址

**设备**: 路由器

#### 2. 数据链路层（Data Link Layer）

**作用**: 同一网络内可靠传输

**功能**:
- MAC 地址寻址
- 帧同步
- 差错控制
- 流量控制

**协议**: 以太网、Wi-Fi、PPP

**设备**: 交换机、网桥

#### 1. 物理层（Physical Layer）

**作用**: 比特流传输

**功能**:
- 定义物理接口
- 定义电压、传输速率
- 定义传输介质

**介质**: 双绞线、光纤、无线电波

**设备**: 集线器、网卡

### 💡 记忆口诀

**从下到上**:
> 物数网传会表应

**谐音记忆**:
> 物（物理层）述（数据链路层）往（网络层）传（传输层）会（会话层）表（表示层）意（应用层）

**英文首字母**:
> **P**lease **D**o **N**ot **T**hrow **S**ausage **P**izza **A**way
> 
> Physical, Data Link, Network, Transport, Session, Presentation, Application

## TCP/IP 四层模型

### 🔥 一句话速答（⭐⭐⭐）
网络接口 → 网际 → 传输 → 应用

### 📊 四层详解

| 层次 | 名称 | 对应 OSI | 协议 | 作用 |
|------|------|---------|------|------|
| **4** | **应用层** | 应用层+表示层+会话层 | HTTP、FTP、DNS、SMTP | 提供应用服务 |
| **3** | **传输层** | 传输层 | TCP、UDP | 端到端通信 |
| **2** | **网际层** | 网络层 | IP、ICMP、ARP | 路由和寻址 |
| **1** | **网络接口层** | 数据链路层+物理层 | 以太网、Wi-Fi | 物理传输 |

### 📝 各层说明

#### 4. 应用层（Application Layer）

**包含**: OSI 的应用层、表示层、会话层

**常见协议**:
- **HTTP/HTTPS**: 网页（端口 80/443）
- **FTP**: 文件传输（端口 21）
- **SMTP**: 邮件发送（端口 25）
- **DNS**: 域名解析（端口 53）
- **SSH**: 安全登录（端口 22）

**示例**: 
```
用户在浏览器输入 www.baidu.com
→ 应用层使用 HTTP 协议请求网页
```

#### 3. 传输层（Transport Layer）

**协议**: TCP、UDP

**作用**:
- 提供端到端通信
- 确保数据可靠传输（TCP）
- 流量控制、拥塞控制

**示例**:
```
HTTP 请求通过 TCP 传输
→ TCP 保证数据不丢失、不重复、按序到达
```

#### 2. 网际层（Internet Layer）

**协议**: IP、ICMP、ARP

**作用**:
- IP 地址寻址
- 路由选择
- 数据包分组转发

**示例**:
```
TCP 段 → 封装成 IP 数据包
→ 根据目的 IP 地址路由到目标主机
```

#### 1. 网络接口层（Network Access Layer）

**包含**: OSI 的数据链路层、物理层

**作用**:
- 定义数据帧格式
- MAC 地址寻址
- 物理传输（电信号、光信号）

**示例**:
```
IP 数据包 → 封装成以太网帧
→ 通过网卡发送到网络
```

### 🆚 OSI vs TCP/IP

| OSI 七层 | TCP/IP 四层 | 说明 |
|---------|------------|------|
| 应用层 | 应用层 | 合并了表示层和会话层 |
| 表示层 | ↑ | - |
| 会话层 | ↑ | - |
| 传输层 | 传输层 | 一致 |
| 网络层 | 网际层 | 一致 |
| 数据链路层 | 网络接口层 | 合并了物理层 |
| 物理层 | ↑ | - |

**为什么 TCP/IP 更实用？**
- OSI 是理论模型，过于复杂
- TCP/IP 是实际应用的标准
- 应用层的三层功能经常混在一起实现

## 五层体系结构

### 🔥 一句话速答（⭐⭐）
物理 → 数据链路 → 网络 → 传输 → 应用

### 📊 五层模型

| 层次 | 名称 | 协议 |
|------|------|------|
| **5** | **应用层** | HTTP、FTP、DNS、SMTP |
| **4** | **传输层** | TCP、UDP |
| **3** | **网络层** | IP、ICMP、ARP |
| **2** | **数据链路层** | 以太网、Wi-Fi |
| **1** | **物理层** | 双绞线、光纤 |

**特点**:
- 是 OSI 和 TCP/IP 的折衷
- 便于教学和理解
- 实际工作中常用的分层方式

## 各层常见协议

### 📊 协议汇总表

| 层次 | 常见协议 |
|------|---------|
| **应用层** | HTTP、HTTPS、FTP、SMTP、POP3、IMAP、DNS、SSH、Telnet |
| **传输层** | TCP、UDP |
| **网络层** | IP (IPv4/IPv6)、ICMP、ARP、RARP、IGMP |
| **数据链路层** | 以太网（Ethernet）、Wi-Fi、PPP、HDLC |
| **物理层** | RJ45、光纤、无线电 |

### 📝 重点协议详解

#### HTTP/HTTPS（应用层）

- **HTTP**: HyperText Transfer Protocol，超文本传输协议
- **HTTPS**: HTTP Secure，加密的 HTTP
- **端口**: 80（HTTP）、443（HTTPS）
- **作用**: 网页浏览

#### DNS（应用层）

- **全称**: Domain Name System，域名系统
- **端口**: 53（UDP）
- **作用**: 域名 → IP 地址
- **示例**: `www.baidu.com` → `220.181.38.148`

#### TCP/UDP（传输层）

| 对比 | TCP | UDP |
|------|-----|-----|
| 连接 | 面向连接 | 无连接 |
| 可靠性 | 可靠 | 不可靠 |
| 速度 | 慢 | 快 |
| 场景 | HTTP、FTP、邮件 | DNS、视频、游戏 |

#### IP（网络层）

- **全称**: Internet Protocol，互联网协议
- **版本**: IPv4（32位）、IPv6（128位）
- **作用**: 寻址和路由
- **示例**: `192.168.1.1`（私有）、`8.8.8.8`（公网）

#### ICMP（网络层）

- **全称**: Internet Control Message Protocol，互联网控制消息协议
- **作用**: 错误报告、网络诊断
- **工具**: `ping`、`traceroute`

#### ARP（网络层）

- **全称**: Address Resolution Protocol，地址解析协议
- **作用**: IP 地址 → MAC 地址
- **示例**: 
  ```
  192.168.1.1 → 00:1A:2B:3C:4D:5E
  ```

## 数据传输流程

### 🔥 一句话速答（⭐⭐⭐）
发送方从上到下层层封装，接收方从下到上层层解封

### 📊 封装与解封装

```
发送方                                         接收方
应用层      [HTTP数据]                    [HTTP数据]        应用层
            ↓                                  ↑
传输层      [TCP头|HTTP数据]            [TCP头|HTTP数据]   传输层
            ↓                                  ↑
网络层      [IP头|TCP头|HTTP数据]      [IP头|TCP头|HTTP数据] 网络层
            ↓                                  ↑
数据链路层  [帧头|IP头|TCP头|HTTP数据|帧尾] [帧头|IP头|TCP头|HTTP数据|帧尾] 数据链路层
            ↓                                  ↑
物理层      [01010101...]                [01010101...]      物理层
            ↓                                  ↑
            ==========  网络传输  ===========
```

### 📝 详细过程

#### 发送方（封装）

**1. 应用层**:
```
数据: "GET /index.html HTTP/1.1"
```

**2. 传输层（添加 TCP 头部）**:
```
[TCP头部] + 数据
源端口: 54321
目的端口: 80
序列号: 1000
```

**3. 网络层（添加 IP 头部）**:
```
[IP头部] + [TCP头部] + 数据
源IP: 192.168.1.100
目的IP: 220.181.38.148
```

**4. 数据链路层（添加帧头和帧尾）**:
```
[帧头] + [IP头部] + [TCP头部] + 数据 + [帧尾]
源MAC: AA:BB:CC:DD:EE:FF
目的MAC: 11:22:33:44:55:66
```

**5. 物理层（转换为比特流）**:
```
01010101010101...
```

#### 接收方（解封装）

**1. 物理层**:
- 接收比特流
- 转换为帧

**2. 数据链路层**:
- 检查帧头和帧尾
- 检查 MAC 地址是否匹配
- 去掉帧头和帧尾
- 传递给网络层

**3. 网络层**:
- 检查 IP 地址是否匹配
- 去掉 IP 头部
- 传递给传输层

**4. 传输层**:
- 检查端口号
- 检查序列号
- 去掉 TCP 头部
- 传递给应用层

**5. 应用层**:
- 处理 HTTP 请求
- 返回响应

### 💡 比喻理解

**寄信比喻**:
```
应用层:    写信内容 "你好"
传输层:    放入内层信封（写收件人姓名）
网络层:    放入中层信封（写收件人地址）
数据链路层: 放入外层信封（写邮局编号）
物理层:    邮递员送信

收信:      一层层拆开信封，最后看到信的内容
```

## URL 访问全流程

### 🔥 一句话速答（⭐⭐⭐⭐）
DNS 解析 → TCP 连接 → HTTP 请求 → 服务器处理 → HTTP 响应 → 渲染页面 → 断开连接

### 📊 完整流程

**示例**: 在浏览器输入 `www.baidu.com`

#### 1. DNS 解析

```
浏览器缓存 → OS 缓存 → 路由器缓存 → ISP DNS → 根DNS → .com DNS → baidu.com DNS
```

**过程**:
1. 检查浏览器 DNS 缓存
2. 检查操作系统 hosts 文件
3. 查询本地 DNS 服务器（递归查询）
4. 本地 DNS 服务器查询根域名服务器（迭代查询）
5. 根域名服务器返回 `.com` 顶级域名服务器地址
6. 查询 `.com` 服务器，返回 `baidu.com` 权威域名服务器
7. 查询 `baidu.com` 服务器，返回 IP 地址 `220.181.38.148`

#### 2. TCP 连接（三次握手）

```
客户端 --[SYN]-------> 服务器
客户端 <--[SYN+ACK]--- 服务器
客户端 --[ACK]-------> 服务器
```

#### 3. 发送 HTTP 请求

```http
GET / HTTP/1.1
Host: www.baidu.com
User-Agent: Mozilla/5.0
Accept: text/html
```

#### 4. 服务器处理请求

- 解析 HTTP 请求
- 查询数据库（如果需要）
- 生成 HTML 页面

#### 5. 返回 HTTP 响应

```http
HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1024

<html>
  <body>百度一下，你就知道</body>
</html>
```

#### 6. 浏览器渲染页面

- 解析 HTML，构建 DOM 树
- 解析 CSS，构建 CSSOM 树
- 合并生成渲染树
- 布局（Layout）
- 绘制（Paint）

#### 7. 断开连接（四次挥手）

```
客户端 --[FIN]-------> 服务器
客户端 <--[ACK]------- 服务器
客户端 <--[FIN]------- 服务器
客户端 --[ACK]-------> 服务器
```

### 📋 涉及的协议

| 步骤 | 协议 | 层次 |
|------|------|------|
| DNS 解析 | DNS (UDP) | 应用层 |
| TCP 连接 | TCP | 传输层 |
| 路由 | IP | 网络层 |
| MAC 寻址 | ARP | 网络层 |
| HTTP 请求/响应 | HTTP | 应用层 |
| 数据传输 | TCP、IP | 传输层、网络层 |
