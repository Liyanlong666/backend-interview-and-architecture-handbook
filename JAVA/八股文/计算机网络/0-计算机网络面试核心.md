## 🎯 高频追问题（面试官最爱问）

### HTTP 相关
1. **301 和 302 的区别？**
   - 301 永久重定向，浏览器会缓存，SEO 友好
   - 302 临时重定向，不缓存

2. **304 是什么场景？**
   - 协商缓存命中，资源未修改，使用本地缓存
   - 通过 `If-None-Match` 或 `If-Modified-Since` 实现

3. **HTTP 2.0 多路复用解决了什么问题？**
   - HTTP 1.x 的队头阻塞问题
   - 一个 TCP 连接可以并发多个请求/响应

4. **HTTP 能传输二进制数据吗？**
   - 可以，通过 `Content-Type: application/octet-stream` 或 `multipart/form-data`

### HTTPS 相关
5. **为什么不全用非对称加密？**
   - 非对称加密计算量大，速度慢
   - 对称加密快，适合大量数据传输

6. **HTTPS 会加密 URL 吗？**
   - URL 路径和参数会加密（在 HTTP 头部）
   - 但域名在 SSL 握手时会暴露（Server Name Indication）

7. **HTTPS 能抓包吗？**
   - 可以，通过中间人证书（如 Fiddler/Charles）
   - 需要客户端信任伪造的 CA 证书

### TCP 相关
8. **为什么 TIME_WAIT 是 2MSL？**
   - MSL = Maximum Segment Lifetime（最大报文生存时间）
   - 确保最后的 ACK 能到达对方 + 让旧连接的包在网络中消失

9. **SYN 洪水攻击是什么？怎么防？**
   - 攻击者发送大量 SYN 不回 ACK，耗尽服务器资源
   - 防御：SYN Cookie、增加半连接队列、防火墙限流

10. **TCP 和 UDP 的区别？**
    - TCP：面向连接、可靠、有序、慢、适合文件传输/HTTP
    - UDP：无连接、不可靠、快、适合视频/游戏/DNS

## 💡 记忆技巧

### HTTP 状态码记忆口诀
- **1xx**："收到继续" （信息）
- **2xx**："办成了" （成功）
- **200**："一切安好"
- **201**："新建完成"
- **204**："办完了没料"（No Content）
- **3xx**："去那边" （重定向）
- **301**："搬家永久"
- **302**："临时搬家"
- **304**："缓存命中"
- **4xx**："你错了" （客户端错误）
- **400**："报文烂了"
- **401**："先登录"
- **403**："我认得你但不给"
- **404**："地址错了"
- **5xx**："我错了" （服务器错误）
- **500**："服务器炸了"
- **502**："网关挂了"
- **503**："服务不可用"

### TCP 三次握手记忆
**农村喊话版**：
- 老王："老张唉！我是老王，你能听到吗？" (SYN)
- 老张："老王老王，我是老张，我能听到，你能听到吗？" (SYN+ACK)
- 老王："老张，我听到了，我有事跟你说。" (ACK)

**泡妞版**：
- 你："嘿，我是小二，我们可以聊聊吗？" (SYN)
- 美女："嗨，我是小青，可以聊聊。" (SYN+ACK)
- 你："太好了，我们微信上聊吧。" (ACK)

## 🚀 进阶拓展（加分项）

如果面试官追问更深，这些点可以加分：

1. **HTTP/3 的 QUIC 协议**
   - 基于 UDP，解决 TCP 队头阻塞
   - 0-RTT 连接建立

2. **WebSocket vs HTTP**
   - WebSocket 是持久化双向通信协议
   - 基于 HTTP 升级（101 Switching Protocols）

3. **多线程断点续传实现**
   - Range 请求头实现分块下载
   - RandomAccessFile 定位写入

4. **抓包工具使用**
   - Chrome DevTools Network 面板
   - Wireshark、Fiddler、Charles

5. **性能优化**
   - HTTP 2.0 Server Push
   - CDN 加速原理
   - 缓存策略（强缓存 vs 协商缓存）
